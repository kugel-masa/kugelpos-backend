# Data Model Design

## Overview

Defines the data models used by the category promotion feature.

## Entities

### 1. PromotionMasterDocument

**Collection name**: `master_promotion`
**Service**: master-data

```
PromotionMasterDocument
├── _id: ObjectId                    # Auto-generated by MongoDB
├── tenant_id: str                   # Tenant ID
├── promotion_code: str              # Promotion code (unique)
├── promotion_type: str              # Promotion type
├── name: str                        # Display name
├── description: str                 # Description
├── start_datetime: datetime         # Start datetime
├── end_datetime: datetime           # End datetime
├── is_active: bool                  # Active flag
├── is_deleted: bool                 # Soft delete flag
├── detail: object                   # Type-specific promotion detail
│   ├── target_store_codes: list[str]     # Target store codes (empty = all stores)
│   ├── target_category_codes: list[str]  # Target category codes
│   └── discount_rate: float              # Discount rate (%)
├── created_at: datetime             # Created datetime
└── updated_at: datetime             # Updated datetime
```

**Field details**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| tenant_id | str | Yes | Tenant identifier |
| promotion_code | str | Yes | Promotion unique identifier |
| promotion_type | str | Yes | e.g. "category_discount" |
| name | str | Yes | Promotion display name |
| description | str | No | Detailed description |
| start_datetime | datetime | Yes | Effective start datetime |
| end_datetime | datetime | Yes | Effective end datetime |
| is_active | bool | Yes | Active/inactive flag (default: true) |
| is_deleted | bool | Yes | Soft delete flag (default: false) |
| detail | object | No | Type-specific promotion details |

**Category promotion detail**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| target_store_codes | list[str] | No | List of target store codes (empty = applies to all stores) |
| target_category_codes | list[str] | Yes | List of target category codes |
| discount_rate | float | Yes | Discount rate (e.g. 10.0 = 10% off) |

**Indexes**:

| Index | Fields | Unique |
|-------|--------|--------|
| idx_tenant_promotion_code | tenant_id, promotion_code | Yes |
| idx_tenant_type_active | tenant_id, promotion_type, is_active | No |
| idx_tenant_active_datetime | tenant_id, is_active, start_datetime, end_datetime | No |

**Validation rules**:

1. `promotion_code` must be unique within tenant_id
2. `end_datetime` > `start_datetime`
3. `discount_rate` must satisfy 0 < rate <= 100
4. When `promotion_type` is "category_discount", `detail` is required
5. `target_category_codes` must contain at least one element

### 2. DiscountInfo (Extension)

**Location**: `services/commons/src/kugel_common/models/documents/base_tranlog.py`

```
DiscountInfo (Existing + Extension)
├── seq_no: int                      # Sequence number (existing)
├── discount_type: str               # Discount type (existing)
├── discount_value: float            # Discount value (existing)
├── discount_amount: float           # Discount amount (existing)
├── detail: str                      # Detail (existing)
├── promotion_code: str              # Promotion code (new)
└── promotion_type: str              # Promotion type (new)
```

**New fields**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| promotion_code | str | No | Code of the applied promotion |
| promotion_type | str | No | Promotion type |

**Backward compatibility**:
- All new fields are Optional
- Existing transaction logs are not affected

### 3. CartDocument.CartLineItem (Reference)

**Location**: `services/cart/app/models/documents/cart_document.py`

Adds promotion information to the discount info of cart line items.
When adding DiscountInfo to the existing `discounts` field,
the new fields (promotion_code, promotion_type) are set.

## State Transitions

### Promotion Lifecycle

```
[Created] → [Active] ⟷ [Inactive] → [Deleted]
               ↓
          [Expired]
```

| State | is_active | is_deleted | Current time |
|-------|-----------|------------|--------------|
| Active | true | false | start ≤ now ≤ end |
| Inactive (manual) | false | false | - |
| Expired | true | false | now > end |
| Deleted | - | true | - |

**Conditions for an active promotion**:
```
is_active = true
AND is_deleted = false
AND start_datetime <= current_time
AND end_datetime >= current_time
```

## Migration

### New Collection

1. Create `master_promotion` collection
2. Create indexes

### Existing Schema Changes

1. Add new fields to `DiscountInfo` (backward compatible)
   - No impact on existing data
   - New fields are Optional, so no changes to existing code are required

### Data Migration

- No migration of existing data is required
- New fields are only set from new transactions onward
